# CSSwitch v0.3.6

上游版本：SuperJJ007/CSSwitch v0.3.6

本次发布基于上游 v0.3.6 对齐，合入主线更新，并在 fork 侧继续补齐远程 / WSL 使用场景。重点是把上游第三方 API 能力接回来，同时修复远程 Helper、端口切换和配置写入里的几个稳定性问题。

## 上游对齐

- 对齐上游 SuperJJ007/CSSwitch v0.3.6，合入主线版本更新。
- 继承上游 OpenAI Responses provider 预览支持，以及近期 provider 兼容性修复。
- 保留 fork 侧远程 / WSL 目标管理能力，并与上游代码重新整理到同一条发布线上。

## 主要更新

- 加强第三方 API 支持：修复 OpenAI custom / Responses 在远程 Helper 场景下的 key、base_url、model 环境变量注入。
- 修改端口时先拆掉旧链路：本地和远程启动都会先停止旧沙箱 / 旧代理，失败时不落盘、不切换到半启动状态，避免运行链路和配置分裂。
- 远程 SSH / WSL 写入 config 和 API key 改走 Helper stdin，避免敏感内容出现在命令行参数。
- Helper Release 仓库改为自动感应：优先读取环境变量，其次读取构建信息，再从 git origin 推断，不再硬编码旧仓库。
- 修复 WSL / 远程 Helper 内置代理资源缺少 `provider_policy.py` / `anthropic_compat.py` 时，代理启动后立即退出的问题。
- 收紧远程状态和停止流程：sandbox 启动失败会清理刚启动的 proxy，减少残留进程和错误状态。
